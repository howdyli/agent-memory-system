## 系统概述

### 产品定位

Agent Memory System 是一个为 AI Agent 提供的完整记忆管理系统，解决 AI 应用在会话持久化、知识积累和上下文管理方面的核心需求。

### 核心能力

| 能力模块 | 说明 | 应用场景 |
|---------|------|-----------|
| 🧠 **记忆变量** | 轻量级 KV 存储（如 `user_name: "鑫海"`） | 用户偏好、会话状态 |
| 📊 **记忆表** | 动态 Schema 结构化数据存储 | 用户数据表、业务实体 |
| 📝 **记忆片段** | 带 TTL 和向量嵌入的语义化记忆 | 长期知识、经验积累 |
| 🔍 **自动记忆召回** | 基于语义相似性的智能记忆检索 | 上下文注入、相关知识推荐 |
| 📚 **长期记忆管理** | 版本控制、自我改进、审计日志 | 知识演进追踪 |

### 目标用户

- **AI Agent 开发者**：为自定义 Agent 集成记忆能力
- **企业 AI 团队**：构建企业级 AI 应用的知识库
- **研究人员**：LLM 记忆机制研究和实验

## 架构设计

### 整体架构

系统采用 **前后端分离** 的现代 Web 架构，分为三层：

```
┌─────────────────────────────────────────────────────────┐
│                 前端层 (Frontend)                        │
│            React 19 + TypeScript + Vite                 │
│            Port: 3000                                   │
└──────────────────────────┬──────────────────────────────┘
                           │ REST API (JWT Auth)
┌──────────────────────────┴──────────────────────────────┐
│                 后端层 (Backend)                         │
│               FastAPI + Python 3.13                      │
│               Port: 8000                                 │
│  ┌───────────┬───────────┬───────────┬──────────────┐ │
│  │ Variables │  Tables   │ Fragments │ Auto Recall  │ │
│  ├───────────┼───────────┼───────────┼──────────────┤ │
│  │Long-Term  │ LLM       │  Plugin   │  Security    │ │
│  │  Memory   │ Backend   │  System   │  Hardening   │ │
│  └───────────┴───────────┴───────────┴──────────────┘ │
└──────────┬────────────────┬───────────────┬─────────────┘
           │                │               │
    ┌──────▼──────┐  ┌──────▼──────┐ ┌──────▼──────┐
    │   SQLite /  │  │   Redis /   │ │  ChromaDB   │
    │ PostgreSQL  │  │  FakeRedis  │ │  (Vectors)  │
    └─────────────┘  └─────────────┘ └─────────────┘
```

### 设计原则

1. **模块化**：每个记忆类型独立服务，便于扩展和维护
2. **多租户**：数据隔离，支持多用户场景
3. **可扩展**：支持从 SQLite 到 PostgreSQL 的无缝切换
4. **安全优先**：JWT 认证 + PBKDF2 密码哈希 + OWASP Top 10 合规
5. **开发者友好**：完整的 Swagger 文档 + LangChain/MCP 集成

## 核心模块

### 3.1 核心基础设施 (Core)

#### Auth 模块
- **职责**：用户认证和授权
- **技术**：JWT Token + PBKDF2 密码哈希
- **关键文件**：`app/core/auth.py`
- **API**：
  - `POST /api/v1/auth/register` - 用户注册
  - `POST /api/v1/auth/login` - 用户登录
  - `GET /api/v1/auth/me` - 获取当前用户信息

#### DB Client 模块
- **职责**：数据库连接管理
- **支持**：SQLite（开发）/ PostgreSQL（生产）
- **关键文件**：`app/core/db_client.py`
- **特性**：
  - 连接池管理
  - 自动表初始化
  - 多租户数据隔离

#### Redis Client 模块
- **职责**：缓存和会话管理
- **支持**：Redis（生产）/ FakeRedis（开发）
- **关键文件**：`app/core/redis_client.py`
- **用途**：
  - 会话缓存
  - 速率限制
  - 临时数据存储

#### ChromaDB Client 模块
- **职责**：向量数据库管理
- **支持**：ChromaDB（本地）/ Milvus（分布式）
- **关键文件**：`app/core/chromadb_client.py`
- **用途**：
  - 向量嵌入存储
  - 语义搜索索引
  - 相似度计算

### 3.2 记忆管理模块 (Memory Modules)

#### 记忆变量 (Memory Variables)
- **功能**：轻量级 KV 存储
- **服务**：`services/memory_variable_service.py`
- **API 路由**：`api/memory_variables.py`
- **核心能力**：
  - 设置/获取/删除变量
  - 从文本自动抽取变量（LLM）
  - 变量模板渲染

**数据流**：
```
用户输入 → LLM 抽取 → 变量存储 (DB) → 模板渲染 → 输出
```

#### 记忆表 (Memory Tables)
- **功能**：动态 Schema 结构化数据存储
- **服务**：`services/memory_table_service.py`
- **API 路由**：`api/memory_tables.py`
- **核心能力**：
  - 动态创建表（运行时定义 Schema）
  - CRUD 操作
  - 自然语言查询（NL to SQL）

**数据模型**：
```python
Table:
  - name: str
  - schema: Dict[str, str]  # 动态字段定义
  - records: List[Dict]      # 数据记录
```

#### 记忆片段 (Memory Fragments)
- **功能**：带 TTL 和向量嵌入的语义化记忆
- **服务**：`services/memory_fragment_service.py`
- **API 路由**：`api/memory_fragments.py`
- **核心能力**：
  - 创建片段（自动生成向量嵌入）
  - TTL 过期管理
  - 语义搜索（基于向量相似度）

**数据流**：
```
文本输入 → LLM 生成嵌入 → ChromaDB 存储
   ↓
语义搜索 → 向量相似度计算 → 返回相关片段
```

#### 自动召回 (Auto Recall)
- **功能**：基于语义相似性的智能记忆检索
- **服务**：`services/auto_recall_service.py`
- **API 路由**：`api/auto_recall.py`
- **核心能力**：
  - 智能记忆检索（多源：变量 + 表 + 片段）
  - 可配置召回策略
  - 召回统计和分析

**召回策略**：
1. **语义匹配**：基于向量相似度
2. **关键词匹配**：基于文本匹配
3. **时间衰减**： recent memories 优先
4. **权重调整**：用户反馈调整权重

#### 长期记忆 (Long-term Memory)
- **功能**：版本控制、自我改进、审计日志
- **服务**：`services/long_term_memory_service.py`
- **API 路由**：`api/long_term_memory.py`
- **核心能力**：
  - 记忆版本记录
  - 审计日志
  - 用户反馈收集
  - 权重自动调整

## 安全架构

### 6.1 认证与授权

- **JWT Token 认证**：
  - Access Token（15分钟）
  - Refresh Token（7天）
  - 自动刷新机制

- **密码安全**：
  - PBKDF2 哈希（100000 迭代）
  - 盐值随机生成
  - 密码强度验证

- **多租户隔离**：
  - 每个用户数据独立
  - `user_id` 字段强制过滤
  - 防止横向越权

- **SQL 注入防护**：
  - 白名单 + 黑名单 + 模式匹配
  - 参数化查询
  - ORM 自动转义

- **XSS 防护**：
  - HTML 清理
  - 输出编码
  - CSP 策略

### 6.3 速率限制

- **限制规则**：
  - 5次/分钟/端点（默认）
  - Redis 计数器
  - 滑动窗口算法

### 6.4 安全审计

- **审计日志**：
  - 所有写操作记录
  - 登录失败记录
  - 异常访问记录

- **OWASP Top 10 合规**：
  - 90% 合规率

  - 自动漏洞检测

## 数据库设计

### 9.1 核心表结构

#### users 表
```sql
CREATE TABLE users (
  id INTEGER PRIMARY KEY,
  username VARCHAR(50) UNIQUE NOT NULL,
  email VARCHAR(100) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### memory_variables 表
```sql
CREATE TABLE memory_variables (
  id INTEGER PRIMARY KEY,
  user_id INTEGER NOT NULL,
  key VARCHAR(100) NOT NULL,
  value TEXT NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id),
  UNIQUE(user_id, key)
);
```

#### memory_tables 表（元数据）
```sql
CREATE TABLE memory_table_meta (
  id INTEGER PRIMARY KEY,
  user_id INTEGER NOT NULL,
  name VARCHAR(100) NOT NULL,
  schema JSON NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id),
  UNIQUE(user_id, name)
);
```

#### memory_fragments 表
```sql
CREATE TABLE memory_fragments (
  id INTEGER PRIMARY KEY,
  user_id INTEGER NOT NULL,
  content TEXT NOT NULL,
  embedding_id VARCHAR(100),
  ttl INTEGER,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  expires_at TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

#### audit_log 表
```sql
CREATE TABLE audit_log (
  id INTEGER PRIMARY KEY,
  user_id INTEGER NOT NULL,
  action VARCHAR(50) NOT NULL,
  resource VARCHAR(50) NOT NULL,
  resource_id VARCHAR(100),
  details JSON,
  ip_address VARCHAR(45),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

### 9.2 向量数据库设计

#### ChromaDB 集合
- **集合名称**：`user_{user_id}_fragments`
- **维度**：768（默认嵌入模型）
- **元数据字段**：
  - `fragment_id` - 片段 ID
  - `created_at` - 创建时间
  - `ttl` - 过期时间
