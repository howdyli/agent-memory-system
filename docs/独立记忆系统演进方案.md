> **目标定位**：将现有"前后端一体应用"演进为"可嵌入 SDK/库"（类似 LangChain Memory / ChromaDB），开发者可集成到自己的 Agent 应用中。
> **演进策略**：渐进式改造，保留现有代码基础，分阶段重构，全程不破坏现有功能。
> **优先能力**：SDK 独立化、多租户与权限、事件与 Webhook、存储可插拔、监控与可观测。

## 1. 总体设计思路

### 1.1 核心理念

将记忆系统从"单一应用"重构为"分层平台 + 可嵌入 SDK"：

- **SDK 优先**：所有记忆能力通过 SDK 暴露，HTTP API 和前端控制台只是 SDK 之上的"参考实现"。
- **存储无关**：上层业务逻辑面向 `MemoryStore` 抽象接口编程，后端（SQLite/PostgreSQL/ChromaDB/Milvus/Qdrant）通过配置切换。
- **多租户原生**：隔离维度从 `user_id` 升级为 `workspace_id`，API Key / OAuth / RBAC 一等公民。
- **事件驱动**：记忆变更通过事件总线广播，支持 Webhook 实时通知，便于与外部系统联动。
- **可观测**：Prometheus 指标 + OpenTelemetry 链路 + 结构化日志，开箱即用。

### 1.2 设计原则

| 原则 | 说明 | 落地手段 |
|------|------|----------|
| 渐进式改造 | 不破坏现有功能，新旧并存 | 双写、Feature Flag、兼容层 |
| 接口先行 | 先定义抽象接口，再实现具体后端 | Protocol/ABC + 依赖注入 |
| 配置驱动 | 行为通过环境变量/YAML 切换，不改代码 | 集中 `Settings` (pydantic-settings) |
| 单一职责 | 每个模块可独立使用 | SDK 包拆分、可选依赖 extras |
| 可测试性 | 存储抽象便于 Mock | 接口注入 + pytest fixtures |

## 2. 模块设计

### 2.1 SDK 独立化

#### 2.1.1 设计思路：双模式 SDK

参考 ChromaDB 的设计，SDK 支持两种使用模式：

- **HTTP 客户端模式（远程）**：SDK 通过 HTTP 调用独立部署的记忆服务，适用于生产/多租户场景。开发者 `pip install agent-memory-sdk`，传入 `base_url` + `api_key`。
- **嵌入模式（本地）**：SDK 直接在进程内使用存储层（SQLite + ChromaDB），无需部署服务端，适用于开发/单机/原型。开发者 `pip install agent-memory-sdk[embedded]`。

现有 `agent_memory_sdk.py` 是"嵌入模式"的雏形（直接 import 服务层），需重构为通过 `MemoryStore` 抽象接口访问，使其既能嵌入又能 HTTP 远程调用。

#### 2.1.3 Python SDK 关键接口设计

`client.py` 核心接口（参考现有 `agent_memory_sdk.py` 的 `AgentMemoryClient`，保持 API 风格一致）：

```python
# sdk-python/src/agent_memory/client.py
from typing import Optional, Any, Dict, List
from .transport import HttpTransport, EmbeddedTransport
from .api import VariablesAPI, FragmentsAPI, TablesAPI, GraphAPI, RecallAPI, EventsAPI

class MemoryClient:
    """
    Agent Memory 统一客户端。

    用法（HTTP 模式）:
        client = MemoryClient(base_url="https://mem.example.com", api_key="ams_xxx")
        client.remember("user_name", "鑫海")
        ctx = client.recall("鑫海的项目")

    用法（嵌入模式）:
        client = MemoryClient(mode="embedded", db_path="./mem.db")
        client.remember_fragment("喜欢极简设计", importance_score=0.9)
    """

    def __init__(
        self,
        base_url: Optional[str] = None,
        api_key: Optional[str] = None,
        workspace_id: Optional[str] = None,
        mode: str = "http",                 # "http" | "embedded"
        # 嵌入模式参数
        db_path: str = "agent_memory.db",
        vector_backend: str = "chroma",
        token: Optional[str] = None,        # JWT token（兼容旧认证）
    ):
        if mode == "http":
            assert base_url, "HTTP 模式需要 base_url"
            self._transport = HttpTransport(base_url, api_key=api_key, token=token, workspace_id=workspace_id)
        elif mode == "embedded":
            self._transport = EmbeddedTransport(db_path=db_path, vector_backend=vector_backend)
        else:
            raise ValueError(f"未知 mode: {mode}")

        # 初始化各 API 子模块
        self.variables = VariablesAPI(self._transport)
        self.fragments = FragmentsAPI(self._transport)
        self.tables = TablesAPI(self._transport)
        self.graph = GraphAPI(self._transport)
        self.recall = RecallAPI(self._transport)
        self.events = EventsAPI(self._transport)

    # 高层便捷方法（保持与现有 AgentMemoryClient 兼容）
    def remember(self, key: str, value: Any, ttl: Optional[int] = None) -> bool:
        return self.variables.set(key, value, ttl=ttl)

    def recall(self, query: str, top_k: int = 5) -> str:
        return self.recall.auto(query, top_k=top_k).context

    def forget(self, key: str) -> bool:
        return self.variables.delete(key)

    def search(self, query: str, top_k: int = 5, threshold: float = 0.3) -> List[Dict]:
        return self.fragments.semantic_search(query, top_k=top_k, threshold=threshold)

    def get_context(self) -> str:
        return self.recall.context()
```

Transport 抽象接口（实现 HTTP/嵌入双模式的关键）：

```python
# sdk-python/src/agent_memory/transport/base.py
from abc import ABC, abstractmethod
from typing import Any, Dict, Optional

class Transport(ABC):
    """传输层抽象：HTTP 远程 或 嵌入式本地"""

    @abstractmethod
    def request(self, method: str, path: str, json: Optional[dict] = None,
                params: Optional[dict] = None) -> Any: ...

    @abstractmethod
    def request_stream(self, method: str, path: str, json: Optional[dict] = None): ...
```

### 2.2 多租户与权限

#### 2.2.1 现状分析

- 现有隔离：`user_id` 列（每张表都有），`enforce_user_isolation(user_id, resource_user_id)`。
- 认证：JWT HS256，`User` 类含 `user_id/username/email`。
- 缺口：无 workspace/org 概念，无 API Key，无 OAuth，无 RBAC。

现有表增加 `workspace_id` 列（迁移脚本自动 backfill）：

```sql
-- 为所有记忆表添加 workspace_id，默认指向用户的 personal workspace
ALTER TABLE memory_variables   ADD COLUMN workspace_id INTEGER;
ALTER TABLE memory_tables      ADD COLUMN workspace_id INTEGER;
ALTER TABLE memory_fragments   ADD COLUMN workspace_id INTEGER;
ALTER TABLE memory_lifecycle   ADD COLUMN workspace_id INTEGER;
ALTER TABLE graph_entities     ADD COLUMN workspace_id INTEGER;
ALTER TABLE graph_relationships ADD COLUMN workspace_id INTEGER;
ALTER TABLE memory_trace_events ADD COLUMN workspace_id INTEGER;
-- ... 其余记忆表同理

-- 创建索引（workspace_id 成为首要过滤字段）
CREATE INDEX idx_mem_var_ws ON memory_variables(workspace_id, key);
CREATE INDEX idx_mem_frag_ws ON memory_fragments(workspace_id, created_at);
-- ...
```

**迁移策略（平滑过渡）**：
1. 迁移脚本为每个现有 `user` 创建一条 `organization`（personal org）+ 一条 `workspace`（personal workspace）。
2. 将所有现有记录的 `workspace_id` backfill 为该用户的 personal workspace id。
3. 查询逻辑从 `WHERE user_id = ?` 改为 `WHERE workspace_id = ?`（保留 `user_id` 作为创建者审计字段）。
4. `enforce_user_isolation` 升级为 `enforce_workspace_access(user, workspace_id)`，校验用户是否为该 workspace 成员。

```python
    authorization: str = Depends(HTTPBearer(auto_error=False)),
    x_api_key: str = Header(None, alias="X-API-Key"),
) -> User:
    """
    统一认证入口，支持：
    1. X-API-Key 头 → API Key 认证
    2. Authorization: Bearer <jwt> → JWT 认证
    3. （OAuth 登录后也签发 JWT，复用 JWT 流程）
    """
    if x_api_key:
        return await _auth_api_key(x_api_key)
    if authorization:
        return await _auth_jwt(authorization.credentials)
    raise HTTPException(401, "Missing authentication credentials")
```

API Key 认证流程：
1. 请求头 `X-API-Key: ams_live_<32位随机>`
2. 取前缀 `ams_live_` + 前 8 位 → 查 `api_keys.key_prefix`
3. sha256(完整 key) 比对 `key_hash`
4. 加载 `workspace_id` + `scopes`，构造 `User(auth_method="api_key")`
5. 更新 `last_used_at`

OAuth 2.0 流程（Authorization Code + PKCE）：
1. 新增路由 `GET /api/v1/auth/oauth/{provider}/login` → 重定向到 provider
2. `GET /api/v1/auth/oauth/{provider}/callback` → 换 token，创建/关联 `oauth_accounts`，签发 JWT
3. 支持 GitHub / Google / Microsoft（初期）

#### 2.3.1 事件总线设计

新建 `backend/app/core/event_bus.py`，抽象事件总线接口，支持内存和 Redis Pub/Sub 两种实现：

```python
# backend/app/core/event_bus.py（新增）
from abc import ABC, abstractmethod
from typing import Any, Callable, Awaitable
from dataclasses import dataclass, field
from datetime import datetime
import asyncio
import json

@dataclass
class MemoryEvent:
    """记忆事件基类"""
    event_type: str                          # memory.created / memory.recalled / ...
    workspace_id: int
    memory_type: str                         # fragment / variable / table / graph
    memory_id: str
    payload: dict = field(default_factory=dict)
    timestamp: str = field(default_factory=lambda: datetime.utcnow().isoformat())
    trace_id: str = ""                       # OTel trace 串联

class EventBus(ABC):
    """事件总线抽象"""
    @abstractmethod
    async def publish(self, event: MemoryEvent) -> None: ...

    @abstractmethod
    def subscribe(self, event_type: str, handler: Callable[[MemoryEvent], Awaitable[None]]) -> None: ...

class InMemoryEventBus(EventBus):
    """内存事件总线（单进程，开发默认）"""
    def __init__(self):
        self._handlers: dict[str, list] = {}
    async def publish(self, event):
        for h in self._handlers.get(event.event_type, []) + self._handlers.get("*", []):
            asyncio.create_task(h(event))
    def subscribe(self, event_type, handler):
        self._handlers.setdefault(event_type, []).append(handler)

class RedisEventBus(EventBus):
    """Redis Pub/Sub 事件总线（多进程，生产用）"""
    def __init__(self, redis_client):
        self._redis = redis_client
        self._handlers = {}
    async def publish(self, event):
        await self._redis.publish(f"memory_events:{event.workspace_id}",
                                   json.dumps(event.__dict__, default=str))
    # subscribe 在后台 task 中通过 pubsub().subscribe() 监听
```

```python
# backend/app/services/webhook_service.py（新增）
class WebhookService:
    async def deliver(self, event: MemoryEvent):
        hooks = self._list_active_webhooks(event.workspace_id, event.event_type)
        for hook in hooks:
            await self._queue_delivery(hook, event)

    async def _queue_delivery(self, hook, event):
        # 写入 webhook_deliveries（pending）
        # 投递到后台 task
        asyncio.create_task(self._do_deliver(hook, event))

    async def _do_deliver(self, hook, event):
        for attempt in range(5):              # 最多 5 次重试
            try:
                signature = hmac_sha256(hook.secret, event.payload_json)
                resp = await httpx.post(hook.url, json=event.__dict__,
                                         headers={"X-AMS-Signature": signature,
                                                  "X-AMS-Event": event.event_type},
                                         timeout=10)
                if resp.status_code < 300:
                    self._mark_success(...); return
            except Exception:
                pass
            await asyncio.sleep(2 ** attempt)  # 1,2,4,8,16s 退避
        self._mark_failed(...)
```

**渐进迁移策略**（关键：不破坏现有功能）：

#### 2.5.5 健康检查增强

改造 `backend/app/api/health.py`，区分 liveness/readiness，检查依赖组件：

```python
# backend/app/api/health.py（改造）
from fastapi import APIRouter, Response, status
from app.core.store.factory import get_relational_store, get_vector_store
from app.core.redis_client import get_redis_client

router = APIRouter()

@router.get("/health/live")
async def liveness():
    """Liveness — 进程存活即 OK（不检查依赖）"""
    return {"status": "alive"}

@router.get("/health/ready")
async def readiness(response: Response):
    """Readiness — 检查所有依赖组件可用"""
    checks = {}
    all_ok = True

    # 关系型数据库
    try:
        get_relational_store()
        checks["database"] = "ok"
    except Exception as e:
        checks["database"] = f"fail: {e}"
        all_ok = False

    # Redis
    try:
        get_redis_client()._connection.ping()
        checks["redis"] = "ok"
    except Exception:
        checks["redis"] = "degraded (fakeredis)"   # fakeredis 降级不算 fatal

    # 向量库
    try:
        get_vector_store().count("memory_fragments")
        checks["vector_store"] = "ok"
    except Exception as e:
        checks["vector_store"] = f"fail: {e}"
        all_ok = False

    if not all_ok:
        response.status_code = status.HTTP_503_SERVICE_UNAVAILABLE
    return {"status": "ready" if all_ok else "not_ready", "checks": checks}
```

## 3. 技术选型

### 3.1 每个模块的关键技术选型

| 模块 | 选型 | 理由 |
|------|------|------|
| **SDK-Python** | httpx + pydantic 2 + hatchling | httpx 支持 sync/async + HTTP/2；pydantic 与后端一致；hatchling 现代打包 |
| **SDK-TypeScript** | @hey-api/openapi-ts 生成 + 手写封装 | OpenAPI schema 自动同步类型；手写封装提供友好 API |
| **多租户** | workspace 隔离 + API Key (sha256) + OAuth2 AuthCode+PKCE | workspace 是业界标准隔离单元；API Key 简单易用；OAuth 标准化 |
| **RBAC** | 静态角色（owner/admin/member/viewer）+ scopes | 初期静态角色够用，避免过度设计；API Key 用 scopes 细粒度控制 |
| **事件总线** | asyncio 内存 + Redis Pub/Sub | 内存默认零依赖；Redis Pub/Sub 支持多进程；暂不需要 Kafka 重量级 |
| **Webhook** | httpx 异步投递 + HMAC-SHA256 签名 + 指数退避 | 签名保证安全；退避保证可靠 |
| **ORM** | SQLAlchemy 2.0（声明式 + type hints） | Python ORM 事实标准；2.0 支持 dataclass 风格 |
| **迁移** | Alembic | SQLAlchemy 官方迁移工具，autogenerate |
| **向量库** | ChromaDB（默认）/ Milvus（大规模）/ Qdrant（备选） | ChromaDB 零运维；Milvus 已有依赖；Qdrant 性能优 |
| **Metrics** | prometheus-fastapi-instrumentator + prometheus-client | 自动 HTTP 指标 + 手动业务指标 |
| **Tracing** | OpenTelemetry SDK + OTLP exporter | CNCF 标准，后端无关（Jaeger/Tempo/Cloud Trace） |
| **日志** | structlog | 结构化 JSON 日志，与 OTel contextvars 集成 |
| **配置** | pydantic-settings | 类型安全 + .env 支持 + 环境变量覆盖 |

### 4.2 阶段依赖关系

```
Phase 0 (地基) ──────┬──→ Phase 1 (存储) ──┬──→ Phase 2 (多租户) ──→ Phase 3 (SDK)
                     │                      │                         │
                     │                      └──→ Phase 4 (事件) ←─────┘
                     │                                  │
                     └──────────────────→ Phase 5 (可观测) ←──┘
```

- Phase 0 是所有阶段的前提（config/logging/metrics 基础）。
- Phase 1（存储）是 Phase 2/4 的前提（schema 迁移、Store 接口）。
- Phase 2（多租户）是 Phase 3（SDK）的前提（SDK 需要稳定的多租户 API）。
- Phase 4（事件）依赖 Phase 1 + 2，可与 Phase 3 部分并行。
- Phase 5（可观测）依赖 Phase 0，可贯穿各阶段增量推进，最后整合。

### 4.3 风险点和缓解措施

| 风险 | 影响 | 缓解 |
|------|------|------|
| Alembic 迁移破坏现有数据 | 数据丢失 | `alembic stamp head` 标记现有库；迁移前备份；先在测试库验证 |
| workspace_id 迁移 backfill 失败 | 数据隔离错乱 | 分批 backfill；dry-run 模式；保留 user_id 兼容查询 |
| SDK 嵌入模式与 HTTP 模式行为不一致 | 开发者困惑 | 共享同一套 Store 接口实现；集成测试覆盖双模式 |
| 双写过渡期数据不一致 | 功能异常 | Feature Flag 控制；一致性校验脚本；可快速回滚 |
| ChromaDB 1.0 升级 breaking change | 向量搜索失效 | 锁定 `<1.0`；VectorStore 抽象隔离；后续单独评估升级 |
| OAuth provider 政策变更 | 登录失败 | 抽象 OAuthService；支持多 provider 降级 |
| OTel instrumentation 性能开销 | 延迟增加 | 采样率配置（默认 10%）；BatchSpanProcessor 异步导出 |
| 前端 api.ts 替换为 SDK 引入回归 | 前端功能异常 | 渐进替换（先新功能用 SDK，旧 API 保留）；充分测试 |

### 5.1 优先级排序与理由

```
Phase 0 (地基)     ████          3-5 天    — 必须最先，所有后续依赖
Phase 1 (存储)     ████████      8-12 天   — 基础设施，schema 迁移前置
Phase 2 (多租户)   █████████     10-15 天  — 商业化必需，SDK 前置
Phase 3 (SDK)      ██████████    12-18 天  — 核心交付物，依赖 1+2
Phase 4 (事件)     ███████       8-12 天   — 可与 Phase 3 并行后半段
Phase 5 (可观测)   █████         6-10 天   — 贯穿增量，最后整合
```

**优先级逻辑**：
1. Phase 0/1 是技术地基，不做后续都受阻。
2. Phase 2（多租户）优先于 Phase 3（SDK），因为 SDK 必须支持多租户 API，认证模型要先稳定。
3. Phase 4（事件）相对独立，可在 Phase 3 后半段并行。
4. Phase 5（可观测）Phase 0 已做基础，最后完善链路和面板。

### SDK 使用示例

**Python（HTTP 模式）**：
```python
from agent_memory import MemoryClient

client = MemoryClient(base_url="https://mem.example.com", api_key="ams_live_xxx")
client.remember("user_name", "鑫海")
client.remember_fragment("喜欢极简设计", importance_score=0.9)
ctx = client.recall("鑫海的项目")
print(ctx)

# 事件订阅
client.events.subscribe("memory.created", lambda e: print(f"新记忆: {e.memory_id}"))
```

**Python（嵌入模式）**：
```python
from agent_memory import MemoryClient

client = MemoryClient(mode="embedded", db_path="./my_agent.db")
client.remember("user_name", "鑫海")
# 无需部署服务端，进程内直接存储
```

**TypeScript**：
```typescript
import { MemoryClient } from '@agent-memory/sdk';

const client = new MemoryClient({ baseUrl: 'https://mem.example.com', apiKey: 'ams_live_xxx' });
await client.remember('user_name', '鑫海');
const ctx = await client.recall('鑫海的项目');
console.log(ctx);
```
